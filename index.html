索引

如果把数据顺序存储在磁盘上，那查找一条数据需要把每条数据都放进内存里，遍历一次。然而内存是比较小的，硬盘里的数据是相对比较多的。所以这种方式的代价是很大的，IO次数很多，速度很慢。

要解决两个问题：
1.搜索速度要快，最好能<O(n)
2.IO次数要尽量少

通过搜索树的结构来减少搜索次数的想法很容易被想到，可以解决第一个问题。
为了解决第二个问题，同时还需要树上的每个结点大小都不太大，可以放进内存里，这样就能减少IO的次数，提高速度。

B+树就可以用来解决这个问题。在磁盘上存储的内容逻辑上就是一颗b+树。叶子结点是具体的数据存储的空间。非叶子结点中存储有指向下一层结点的指针。

聚集索引clustered index 叶子结点中存储的是具体的数据，每一张表有且只有一个聚集索引，可能是多个字段的联合索引。
非聚集索引secondary index 叶子结点中存储的是聚集索引的key值。非聚集索引是在另一片存储中建立的索引，索引的叶子结点里存储的是聚集索引的key
联合索引：左前缀原则。当多个字段建立索引时，例在a b c 上建立联合索引，则where里写前面的字段时，会用到索引，例select * from tab where a = 'a' and b = 'b';
顺序不对了，则索引不起作用了，例select * from tab where b = 'b' and c = 'c'

TCP报头

和三次握手四次挥手相关的有4个：
1.Seq号，每个TCP报文都有的一个序号
2.flag：8位标志位，每一位标志一类报文，标识了当前这种报文对应的信息有效，包括SYN/FIN/ACK/RST/PUSH/URG/ECE/CWR。
握手挥手中用到了SYN、ACK、FIN，
SYN为1时，标识该包为请求链接的包
ACK为1时，标识TCP头部中的ACK编号有效
FIN为1时，标识该包为断开链接的请求包
3.ACK编号：当falg中的ACK为1时有效，如果正确，则应该是刚刚收到的对方发来的包的Seq+1，发回给发送方，发送方即可验证刚刚发的信息是不是被对方收到了。

TCP3次握手

SYN（Synchronize Sequence Numbers同步序列编号）：发送给对方的请求建连接的flag位
ACK（Acknowledgment确认符）:响应对方发送的Seq号的flag位，以及ACK编号，两个东西。flag位中标识了表示ACK序号有效，ACK序号为收到报文的Seq+1。用于告诉对方我收到了刚刚的报文
FIN（Finish?）：发送发给对方用于断开连接的flag位

发送方 接收 flag置为1的位 发送的Seg/ACK
客户端 \ SYN Seq = X
服务端 接收对方的SYN Seq = X SYN+ACK Seq = Y, ACK = X+1
客户端 接收ACK验证是否为X+1 ACK Seq = X+1, ACK = Y+1

数据传输

主动 报文
发送方发 Seq+ACK
接收方发 ACK

想要发送就要发Seq序列号

TCP4次挥手

发送方 接收 flag置为1的位 发送内容
客户端 \ FIN+ACK Seq = M ,ACK = Y+1(若上一次接收的seq是Y)
服务端 接收Seq = X+2 ACK ACK = M+1
服务端 也发送一个断开连接的包 FIN Seq = N
客户端 接收ACK验证是否为X+3 ACK ACK = N+1

Leetcode TwoSum

Posted on 2019-04-13

题目描述:

输入一个整数序列nums，和一个target整数，希望找到这个序列中，两个和为target的下标。

例1：
输入3,2,4;6
输出1,2
解释：目标是6，2+4=0，2的下标是1，4的下标2

例2：
输入2,3,5,7;9
输出0,3

思路

1.暴力2层遍历，两两加和，遇到为target的则返回O(n^2)
2.遍历一次O(n)，用target-nums[i],得到的值，再去nums里查找，查找最快的算法是用hash结构，O(1).整体的时间复杂度就是O(n)

编程实现

原来传入的nums是数组或者vector的，不能直接用hash查找。要转储成hash的结构才可以。
因为我们要返回的是下标，所以存储时需要用pair,最合适的结构应该是hashmap,但是c++STL里没有hashmap,有两个选择：
1.自己建一个hash，用二维数组，一列存值，一列存以前的下标，用线性探测解决冲突，一层用值做索引。
2.用一个退而求其次的方法，比如map，map底层是红黑树，查找时间复杂度是O(logn).整体时间复杂度是O(nlogn)

考虑编程时间的限制，选择了第二种方式

思路：用补码的定义，找到其去掉符号位的补码对应的正数有多少个1，再+1个符号位.

借机复习一下原码 反码 补码
正数：原码=反码=补码
 正数122的原码，补码，反码都是：00….01111010（如果是c语言中int类型，则一共32位）

负数：
 原码：对其绝对值原码，最高位改为1
 反码：对其绝对值原码，逐位取反
 补码：
  描述1：与其绝对值直接相加等于0的编码。n+1位二进制负数A（一共n+1位，其中最高位是符号位，数据只有n位）的补码为(10000…(一共n+1位，n个0)-A的绝对值)，即$2^{n+1}-{|A|}$的原码（或者拿更高位数的10000..0来减，得到的结果在后n位都是一样的），再加上最高位的符号位1。这个数与其绝对值直接做二进制加法等于1000…0（n+2位，最高位为1，但是溢出了所以保存不下来，剩余的n+1位为0），由于位数溢出了一位，其结果就是0，满足了描述1的第一句话。
  在《计算机系统概论》里详细阐述了计算机算数逻辑单元ALU做简单二进制加法很容易，不希望其关心操作数到底是正是负。以上的构造方式，计算起来非常方便，可以不对负数进行单独的处理，可直接用其补码参与运算，进位时直接省掉了最高位。而且可以省掉反码里两个不同的串用来表示同一个数0，其中的的-0：100000…0，这个数在补码中是这么多位能表示的最小负数。详细讲解可以看《计算机系统概论》中第2.3节。
  描述2：绝对值的反码+1。|A|加上|A|的反码，结果是一串1，再加上1就得到了之前所说的，多一位的10000…0，所以|A|+(|A|反码+1)=10000…0，(|A|反码+1)即为A的补码
 负数-122的
 原码：10…01111010 (32位，最高位为1，其余和正数122相同)
 反码：11…10000101 (其绝对值原码逐位取反，或，其原码除最高位外逐位取反)
 补码：11…10000110 (反码+1)
