Leetcode TwoSum 两数和

Posted on 2019-04-13

题目描述:

输入一个整数序列nums，和一个target整数，希望找到这个序列中，两个和为target的下标。

例1：
输入3,2,4;6
输出1,2
解释：目标是6，2+4=0，2的下标是1，4的下标2

例2：
输入2,3,5,7;9
输出0,3

思路

1.暴力2层遍历，两两加和，遇到为target的则返回O(n^2)
2.遍历一次O(n)，用target-nums[i],得到的值，再去nums里查找，查找最快的算法是用hash结构，O(1).整体的时间复杂度就是O(n)

编程实现

原来传入的nums是数组或者vector的，不能直接用hash查找。要转储成hash的结构才可以。
因为我们要返回的是下标，所以存储时需要用pair,最合适的结构应该是hashmap,但是c++STL里没有hashmap,有两个选择：
1.自己建一个hash，用二维数组，一列存值，一列存以前的下标，用线性探测解决冲突，一层用值做索引。
2.用一个退而求其次的方法，比如map，map底层是红黑树，查找时间复杂度是O(logn).整体时间复杂度是O(nlogn)

考虑编程时间的限制，选择了第二种方式

剑指Offer-二进制int中1的个数

Posted on 2019-03-30

题目描述

思路

思路：用补码的定义，找到其去掉符号位的补码对应的正数有多少个1，再+1个符号位.

借机复习一下原码 反码 补码
正数：原码=反码=补码
 正数122的原码，补码，反码都是：00….01111010（如果是c语言中int类型，则一共32位）

负数：
 原码：对其绝对值原码，最高位改为1
 反码：对其绝对值原码，逐位取反
 补码：
  描述1：与其绝对值直接相加等于0的编码。n+1位二进制负数A（一共n+1位，其中最高位是符号位，数据只有n位）的补码为(10000…(一共n+1位，n个0)-A的绝对值)，即$2^{n+1}-{|A|}$的原码（或者拿更高位数的10000..0来减，得到的结果在后n位都是一样的），再加上最高位的符号位1。这个数与其绝对值直接做二进制加法等于1000…0（n+2位，最高位为1，但是溢出了所以保存不下来，剩余的n+1位为0），由于位数溢出了一位，其结果就是0，满足了描述1的第一句话。
  在《计算机系统概论》里详细阐述了计算机算数逻辑单元ALU做简单二进制加法很容易，不希望其关心操作数到底是正是负。以上的构造方式，计算起来非常方便，可以不对负数进行单独的处理，可直接用其补码参与运算，进位时直接省掉了最高位。而且可以省掉反码里两个不同的串用来表示同一个数0，其中的的-0：100000…0，这个数在补码中是这么多位能表示的最小负数。详细讲解可以看《计算机系统概论》中第2.3节。
  描述2：绝对值的反码+1。|A|加上|A|的反码，结果是一串1，再加上1就得到了之前所说的，多一位的10000…0，所以|A|+(|A|反码+1)=10000…0，(|A|反码+1)即为A的补码
 负数-122的
 原码：10…01111010 (32位，最高位为1，其余和正数122相同)
 反码：11…10000101 (其绝对值原码逐位取反，或，其原码除最高位外逐位取反)
 补码：11…10000110 (反码+1)

三囚徒问题，三门问题浅析

Posted on 2019-01-16

前言

最近在学习《统计推断》(Statistical Interferent)时，在讨论条件概率的部分，提到了一个经典的问题：三囚徒问题。其中的一些前提条件数上没有完全阐述清楚，也没有给出严格的用定义来推到的过程。所以产生了很多疑问。在这里对这个题目未给出的假设条件进行补充（依据书上给出的答案），并且讨论缺少这个假设条件时的情况。

使用到的公式

条件概率公式：$P(A|B) = \frac{P(A∩B)}{P(B)}$
解释：在B发生的条件下A发生的概率为AB同时发生的概率，除以B发生的概率。

乘法公式：P(A∩B) = P(A|B)P(B)

三囚徒问题

三囚徒问题描述

有三个本来要处死的囚犯A、B、C，监狱长决定要随机赦免其中之一，即三人被赦免的概率是相同的，都为$\frac{1}{3}$，但不告诉大家赦免的人是谁.囚犯A去问监狱长他自己是否被赦免，监狱长拒绝回答。于是A转变提问方式，他又问B和C谁会被处死。

监狱长认为每个人有$\frac{1}{3}$的概率被赦免，所以BC中必然会处死一人，所以他没有透露任何关于囚徒A是否会被处死的信息。

囚徒A认为B一定会被处死，而他自己A和C中只有一人会被处死，他被赦免的概率就上升到了$\frac{1}{2}$.
谁是正确的？

补充基本假设条件

监狱长不能说A是否会被处死，只能说B被处死或C被处死，所以只有两个选择，并且两个选择的概率是等可能的，都为$\frac{1}{2}$。
这里A存活的概率和C存活的概率我们可以都求，用来类比三门问题。

符号表示

A事件：A被赦免
B事件：B被赦免
C事件：C被赦免
P(A)=P(B)=P(C)=$\frac{1}{3}$，且P(i∩j)=0
W：事件（监狱长说B被处死），P(W)=$\frac{1}{2}$
求P(A|W)：在监狱长说B被处死的情况下，A被赦免的概率.
求P(C|W)：在监狱长说B被处死的情况下，C被赦免的概率.

求解过程

$P(A|W) = \frac{P(A∩W)}{P(W)}$
= $\frac{P(W|A)P(A)}{P(W)}$
= $\frac{\frac{1}{2} *\frac{1}{3}}{\frac{1}{2}}$
= $\frac{1}{3}$
备注：由乘法公式：P(A∩W) = P(W|A)P(A)，P(W|A)=$\frac{1}{2}$
因为A被赦免的情况下，法官说B或C都可以，且之前假设了说B说C是等概率的，所以这里是$\frac{1}{2}$

$P(C|W) = \frac{P(C∩W)}{P(W)}$
=$\frac{P(W|C)P(C)}{P(W)}$
=$\frac{1 * \frac{1}{3}}{\frac{1}{2}}$
=$\frac{2}{3}$
备注：P(W|C) = 1,因为C被赦免的情况下，法官不能透露A的信息，所以一定会说B被处死

三囚徒问题结论

在监狱长说B会被处死的情况下，A被赦免的概率不变，还是1/3，但C被赦免的概率升高到了2/3.

不补充”说B死””说C死”等概率的条件

这种情况下不能保证”说B死””说C死”的概率都为$\frac{1}{2}$，这样就不能直接用公式代入条件概率了。因为P(W)不能用具体的数值表达出来。

三门问题

问题描述

三门问题源自于一个电视节目游戏，具体历史可以查看百度三门问题，这里只阐述问题本身。游戏由一个主持人、一个猜奖者、三扇门、两只羊和一辆车构成。
游戏开始时，三扇门关闭，每扇门背后有一个物品，一只羊或一辆车。主持人知道哪扇门后是车。猜奖者随意说出一扇门的编号。此时主持人打开一扇后面是羊的门。问猜奖者要不要换门，猜奖者决定最后要开的门后，若是车，则车属于猜奖者。
请问，猜奖者要不要换门？即换门之后获奖概率是否会提高？

基本假设及分析解释

1.每扇门后有车的概率相同，都为$\frac{1}{3}$，有且只有一扇门后有车，另外两扇门后是羊.

2.主持人只能打开是羊的门，并且不会打开猜奖者猜测的那扇门。
即若猜奖者猜门1，车实际在门2，则主持人只能打开门3.若猜奖者猜门1，车实际在门3，则主持人只能打开门2.猜奖者猜门1，车实际在门1，则主持人以$\frac{1}{2}$打开门2或门3.
前两种情况发生的概率是相等的，分别对应着打开门2和门3的概率也相等，最后一种中两门也是等概率的，所以各自是$\frac{1}{2}$

3.我们可以通过求解两个概率：1) 保持原来的选择（主持人不能透露信息的那扇门），2) 换门（排除原本选择和主持人打开的门，剩下的那扇门）
二者门后有车的概率，进行比较，若1)高则不换们，若2)高则换门。
由此则转换为了和三囚徒问题相同的前提条件和求解目标。

符号表达及计算结果

猜奖者选择了门A
A事件：A门后有车
B事件：B门后有车
C事件：C门后有车
P(A)=P(B)=P(C)=$\frac{1}{3}$，且P(i∩j)=0
W事件：主持人打开了门B，P(W)=$\frac{1}{2}$（基本假设及分析解释2）
求P(A|W)：在主持人打开了门B的情况下，A门后是车的概率.
求P(C|W)：在主持人打开了门B的情况下，C门后是车的概率.

求解过程同三囚徒问题，P(A|W) = $\frac{1}{3}$，P(C|W) = $\frac{2}{3}$.

三门问题结论

三门问题中换门会提高中奖的概率。

骰子问题（推广到多个选择）

问题描述

一个均匀的6面骰子，每个面朝上的概率完全相等为$\frac{1}{6}$，投掷一次，两个人，一个猜，一个看。
投掷出结果后：

条件a：B先说“不是某一点”，某一点在{1,2,3,4,5,6}中选择，然后A猜测。A猜中的概率记为P(Aa).更换猜的点，猜中的概率为P(Ba).

条件b：A先猜，A猜完后告诉B。B避开A猜测的结果，告诉他“不是某一个点”。假设A猜的是1，则“某一个点”在{2,3,4,5,6}中选择。A猜中的概率记为P(Ab).（即避免直接告诉A骰子的结果，比如A猜的1，结果不是1，则不能说“不是1”）.更换猜的点，猜中的概率为P(Bb).

条件c：A先猜，A猜完后告诉B。B避开A猜测的结果，可以说“不是某几个点”。（即避免直接告诉A骰子的结果，若结果是1，则不能说“不是23456”）A猜中的概率记为P(Ac).更换猜的点，猜中的概率为P(Bc).

求解

P(Aa) = $\frac{1}{5}$
P(Ba) = $\frac{1}{5}$

P(Ab) = $\frac{1}{6}$
P(Bb) = $\frac{1}{6} * \frac{5}{4}$ = $\frac{5}{24}$

P(Ac) = $\frac{1}{6}$
