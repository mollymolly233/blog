TCP报头

和三次握手四次挥手相关的有4个：
1.Seq号，每个TCP报文都有的一个序号
2.flag：8位标志位，每一位标志一类报文，标识了当前这种报文对应的信息有效，包括SYN/FIN/ACK/RST/PUSH/URG/ECE/CWR。
握手挥手中用到了SYN、ACK、FIN，
SYN为1时，标识该包为请求链接的包
ACK为1时，标识TCP头部中的ACK编号有效
FIN为1时，标识该包为断开链接的请求包
3.ACK编号：当falg中的ACK为1时有效，如果正确，则应该是刚刚收到的对方发来的包的Seq+1，发回给发送方，发送方即可验证刚刚发的信息是不是被对方收到了。

TCP3次握手

SYN（Synchronize Sequence Numbers同步序列编号）：发送给对方的请求建连接的flag位
ACK（Acknowledgment确认符）:响应对方发送的Seq号的flag位，以及ACK编号，两个东西。flag位中标识了表示ACK序号有效，ACK序号为收到报文的Seq+1。用于告诉对方我收到了刚刚的报文
FIN（Finish?）：发送发给对方用于断开连接的flag位

发送方 接收 flag置为1的位 发送的Seg/ACK
客户端 \ SYN Seq = X
服务端 接收对方的SYN Seq = X SYN+ACK Seq = Y, ACK = X+1
客户端 接收ACK验证是否为X+1 ACK Seq = X+1, ACK = Y+1

数据传输

主动 报文
发送方发 Seq+ACK
接收方发 ACK

想要发送就要发Seq序列号

TCP4次挥手

发送方 接收 flag置为1的位 发送内容
客户端 \ FIN+ACK Seq = M ,ACK = Y+1(若上一次接收的seq是Y)
服务端 接收Seq = X+2 ACK ACK = M+1
服务端 也发送一个断开连接的包 FIN Seq = N
客户端 接收ACK验证是否为X+3 ACK ACK = N+1
