GitHub+Nodejs+Hexo搭建个人博客

Posted on 2018-12-11

利用git版本控制器，nodejs 基于 Chrome V8 引擎的 JavaScript 运行环境 ，hexo前端框架
nodejs是一个让js运行在服务器端的环境

原始参考经验：

https://www.cnblogs.com/fengxiongZz/p/7707219.html

样式更改：

解决：https://blog.csdn.net/Hoshea_chx/article/details/78826689
总结：在blog目录下，通过git引入主题，再更改hexo的配置文件_config.yml的默认主题。
更改scheme，在theme目录下对应的theme里的配置文件_config.yml更改

报错：

1.

FATAL bad indentation of a mapping entry
解决：https://blog.csdn.net/zoeyyeoz/article/details/51143613
总结原因：hexo的配置文件是yml格式的，key和value之间用:隔开，但是，冒号之后必须有一个空格。

2.

Unexpected end of JSON input while parsing near
解决：https://blog.csdn.net/yangluo1683/article/details/79788345
总结：缓存未清除

git上没有样式，本地有：

解决：https://segmentfault.com/q/1010000002924963
总结：因为建立的repository是blog，而不是git用户名.github.io，所以blog的网址是https://mollymolly233.github.io/blog/，多了一层目录blog，而不是https://mollymolly233.github.io
所以需要更改hexo的配置文件中url和root的路径，才能使hexo正确地找到css文件。
